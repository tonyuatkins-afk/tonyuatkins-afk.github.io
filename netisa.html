NetISA

Open-source TLS 1.3 and WiFi for vintage ISA PCs.

Phase 0: Hardware designed. CPLD logic validated (160/160 tests). Parts ordered, awaiting prototype build.
Phase 1: DOS software stack complete. v1 ESP32-S3 firmware complete. Cathode v0.2 document browser with streaming HTML parser, Discord client, and Claude AI client running in DOSBox-X. All code quality-gated with iterative adversarial review.

NetISA is an 8/16-bit ISA expansion card that gives IBM PC/XT, AT, 386, and 486 systems a first-class path to the modern internet. A Microchip ATF1508AS CPLD handles ISA bus timing deterministically. An Espressif ESP32-S3 handles WiFi, TLS 1.3, and the full TCP/IP stack using hardware-accelerated AES, SHA, RSA, and ECC. The retro PC sees a register-mapped coprocessor and talks to it through a small DOS TSR. No proxy box. No serial bottleneck. No software crypto on the retro CPU.

What this unlocks

  • HTTPS browsing from DOS via Cathode (built-in text-mode browser), Lynx, Links, or Arachne against real TLS 1.3 sites.
  • Gmail, Outlook, Fastmail from Pine or Pegasus via IMAPS and SMTPS.
  • git clone over HTTPS on a 486 against GitHub.
  • Discord, Mastodon, Bluesky, Matrix via REST APIs, plus IRC over TLS for Libera.Chat.
  • Claude AI on a vintage PC via the Anthropic API over TLS 1.3, with agent mode for running DOS commands.
  • Dropbox, OneDrive, Nextcloud sync via WebDAV over HTTPS.
  • MQTT over TLS for smart home publishing from an 8088.

DOS software

The full DOS software stack is built and tested in DOSBox-X. All code targets 8088 real mode (OpenWatcom 2.0, small memory model) and runs on anything from a 4.77 MHz XT to a Pentium.

  • NETISA.COM — 678-byte TSR that hooks INT 63h. Under 2 KB resident. Provides the software interface between DOS applications and the card.
  • NETISA.EXE — Full-screen launcher with WiFi network scanning, signal strength bars, password entry, card status display, and system information (CPU detection, FPU, DOS version, free RAM).
  • Screen library — Shared VGA text buffer rendering engine. Direct writes to 0xB800:0000 with CP437 box drawing, design-language color palette, and keyboard input via INT 16h.
  • INT 63h API — Complete C wrapper library (netisa.h) matching the architecture spec. Eight function groups covering system, WiFi, DNS, sessions, certificates, crypto, events, and diagnostics.
  • Stub layer — Fake data layer for DOSBox-X testing without hardware. Simulates WiFi networks, connection states, version info, and BIOS tick-based delays.

Cathode v0.2: text-mode document browser

Cathode is a text-mode document browser for DOS. It fetches real HTML pages over HTTPS via the NetISA card, parses them through a streaming HTML parser, and renders structured content in 80×25 text mode. Cathode excels at articles, documentation, wikis, and directory pages.

  • Streaming HTML parser — headings, paragraphs, links, bold/italic/code, ordered and unordered lists, blockquotes, preformatted text, horizontal rules, HTML entities, and image alt text
  • HTTP/1.0 fetch with redirect following, UTF-8 to CP437 character mapping, and URL resolution for relative links
  • Find on page (Ctrl+F) with match highlighting and N/Shift+N cycling
  • Bookmarks with disk persistence (Ctrl+D to save, Ctrl+B to view)
  • 200-row scrollback, scroll bar, link navigation via Tab/Shift+Tab, back/forward history
  • CP437 block-art CATHODE logo on start page
  • 38 KB EXE. Quality-gated: 5 rounds of adversarial code review, 34 findings resolved

Discord client

A text-mode Discord client demonstrating the card's ability to connect vintage PCs to modern chat platforms. Runs with stub data for DOSBox-X testing.

  • 6 channels with per-channel far-heap message storage
  • Word-wrapped messages with author colors
  • Compose bar with cursor editing, Tab-cycling focus
  • Timed fake message injection (~5 second intervals)
  • PgUp/PgDn scrolling with position tracking
  • Quality-gated: 6 rounds of adversarial review

Claude for DOS

A text-mode Claude AI chat client that talks to the Anthropic API over TLS 1.3. Includes a 3-mode agent system for running DOS commands directly from the conversation. Runs with stub responses for DOSBox-X testing.

  • Chat, Ask, and Auto agent modes (F4 to cycle)
  • Far-heap message pool for small memory model
  • Word-wrapped chat rendering with newline support
  • 3-line compose area with cursor editing
  • [EXEC] tag parsing for command execution with output capture
  • Iterative agent loop with 5-deep safety cap (no recursive stack growth)
  • Quality-gated: 2 rounds of adversarial review

v1 Firmware

The v1 ESP32-S3 firmware replaces the Phase 0 loopback with production networking. It preserves the PSTROBE ISR and register file architecture — the ISA bus interface is unchanged.

  • WiFi manager: APSTA mode with auto-connect from saved credentials, network scanning, and a built-in setup AP ("NetISA-Setup")
  • HTTP client: Up to 4 concurrent TLS sessions using the ESP-IDF certificate bundle for public CA coverage
  • HTML parser: State machine that converts HTML to CP437 cell streams for the Cathode browser
  • Web config: Embedded HTTP server at 192.168.4.1 with WiFi setup, OTA firmware updates (authenticated), and status monitoring
  • Command dispatch: Maps all 8 INT 63h function groups to handlers via FreeRTOS queue between ISR (Core 0) and task (Core 1)
  • Security: Admin key authentication on all destructive endpoints, OTA password protection, content validation, memory barriers on cross-core shared state
  • Quality-gated: 8 rounds of adversarial review — fixed cross-core race conditions, register protocol bugs, security vulnerabilities, and resource leaks

Specs

Bus interface8/16-bit ISA (XT through 486)
Bus logicMicrochip ATF1508AS CPLD, TQFP-100, 128 macrocells, native 5 V, 10 ns pin-to-pin
ProcessorESP32-S3-WROOM-1U-N8R8 (dual-core Xtensa LX7, 8 MB flash, 8 MB PSRAM)
WiFi802.11 b/g/n, external bracket-mount antenna for metal cases
TLS1.3 with hardware AES-128/256, SHA-256/384/512, RSA-4096, ECC, hardware RNG
EthernetWiznet W5500, 8 hardware TCP sockets, 10/100 Mbps (v1.5, pads on v1 board)
Host CPUAny x86: 8088 4.77 MHz through Pentium+
DOS driver678-byte TSR, < 2 KB resident, INT 63h API
Packet driverPC/TCP compatible (mTCP, WATTCP applications)
I/O decodeFull 16-bit, no aliasing on AT+ systems
ConfigurationPhysical jumpers for base address, IRQ, safe mode. No Plug-and-Play.
LicenseMIT (software) / CERN-OHL-P (hardware)

Design verification

  • 160 of 160 Verilog testbench tests passing (iverilog), covering address decode, IOCHRDY wait states, watchdog timeout, IRQ state machine retrigger, back-to-back cycles, mid-cycle reset, status flag merge, and alias rejection.
  • 95 of 128 CPLD macrocells used, 61 of 84 pins, fits EPM7128STC100-15 clean on Quartus II 13.0sp1.
  • Timing slack: +39.5 ns setup, +5.0 ns hold, +26.25 ns minimum pulse width at 16 MHz. Zero total negative slack.
  • Zero bus contention events across the entire simulation suite.
  • Lint clean on iverilog -Wall.

Looking ahead

v1 ships as a DOS and Windows 3.x peripheral using Session Mode (the card owns TCP/IP and TLS, host talks at the session level). The firmware and CPLD are deliberately architected so v2 can add native drivers for Windows 95/98/NT NDIS, Linux kernel net_device, and NetBSD/FreeBSD, without hardware changes. v2.5 adds Linux kTLS offload, matching the architecture Mellanox and Chelsio use for their datacenter TLS-capable NICs, brought to a 486.
